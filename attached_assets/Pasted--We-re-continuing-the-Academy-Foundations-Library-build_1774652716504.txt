We're continuing the Academy / Foundations Library build on the "feature/academy-foundations-library" branch. Make sure you're on that branch. This is Part 2 of 4 — member-facing API routes and pages.

=== MEMBER API ROUTES (8 routes) ===

All routes use resolveUserFromSession() for multi-tenancy, consistent with existing codebase patterns.

1. GET /api/member/academy/sections
   - Returns all published sections ordered by sortOrder
   - Include lesson count per section
   - Include member's progress: lessons completed / total lessons per section
   - Response shape: { sections: [{ id, title, slug, description, sortOrder, lessonCount, completedCount }] }

2. GET /api/member/academy/sections/[sectionId]/lessons
   - Returns all published lessons in a section, ordered by sortOrder
   - Include member's completion state for each lesson
   - Response shape: { lessons: [{ id, title, slug, youtubeUrl, sortOrder, completed, principleTags }] }

3. GET /api/member/academy/lessons/[lessonId]
   - Returns full lesson detail including:
     - All lesson fields (description, keyTakeaways, actionItems, etc.)
     - All workbook fields with the member's saved responses
     - Member's homework items (if any saved)
     - Completion state
     - Previous and next lesson IDs/slugs for navigation
   - Response shape: { lesson: { ...allFields, workbookFields: [{ id, fieldType, label, placeholderText, sortOrder, config, response }], homework: { homeworkItems }, completed, completedAt, prevLesson: { slug, sectionSlug }, nextLesson: { slug, sectionSlug } } }

4. PUT /api/member/academy/workbook/[fieldId]
   - Body: { response: any } (JSON — the response format depends on field type)
   - Upserts MemberWorkbookResponse for this user + field
   - Returns { success: true }
   - This is called on every auto-save (debounced on frontend)

5. PUT /api/member/academy/lessons/[lessonId]/progress
   - Body: { completed: boolean }
   - Upserts MemberLessonProgress
   - If completed=true, sets completedAt to now
   - If completed=false, clears completedAt
   - Returns { success: true, completed, completedAt }

6. PUT /api/member/academy/lessons/[lessonId]/homework
   - Body: { homeworkItems: [{ label: string, completed: boolean }] }
   - Upserts MemberHomework
   - Returns { success: true }

7. GET /api/member/academy/principles
   - Returns all 17 principles with count of associated lessons
   - Response shape: { principles: [{ slug, name, lessonCount }] }
   - Principle name mapping (hardcoded is fine):
     avatar_clarity → Avatar Clarity
     themes_over_topics → Themes Over Topics
     lead_magnet → Lead Magnet
     arc_attention → ARC Attention
     arc_revelation → ARC Revelation

     connection_language → Connection Language
     approve_the_click → Approve the Click
     title_frameworks → Title Frameworks
     curiosity_bridges → Curiosity Bridges
     binge_architecture → Binge Architecture
     consistency → Consistency
     values_peppering → Values Peppering
     grade_5_language → Grade 5 Language
     story_proof → Story Proof
     show_dont_tell → Show Don't Tell
     studio_setup → Studio Setup

8. GET /api/member/academy/principles/[slug]
   - Returns principle name + all published lessons tagged with that principle slug
   - Searches the principleTags JSON array field
   - Response shape: { principle: { slug, name }, lessons: [{ id, title, slug, sectionTitle, sectionSlug }] }
