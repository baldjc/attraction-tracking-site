  Context                                                                                                                                                    
                                                                                                                                                             
  The admin Foundations Members page (/admin/members) shows a "Tier" column that's manually set via serviceTier on the User model. We want to add Stripe     
  integration so subscription status syncs automatically and admins can see what Stripe plan each member is on.                                              
   
  Currently there is NO Stripe SDK, no API keys, and no webhook handling. The only Stripe touchpoint is payment link URLs stored on ServicePackage records.  
                                                                                                                                                           
  Task 1: Install Stripe SDK                                                                                                                                 
                                                                                                                                                           
  Install the stripe npm package:                                                                                                                            
  npm install stripe
                                                                                                                                                             
  Task 2: Add Environment Variables                                                                                                                        

  Add these to .env (and .env.example):                                                                                                                      
  STRIPE_SECRET_KEY=
  STRIPE_WEBHOOK_SECRET=                                                                                                                                     
                                                                                                                                                           
  Task 3: Prisma Schema Changes
                                                                                                                                                             
  Add these fields to the User model in prisma/schema.prisma:
                                                                                                                                                             
  stripeCustomerId    String?   @unique                                                                                                                    
  stripeSubscriptionId String?                                                                                                                               
  subscriptionStatus   String?   // "active", "past_due", "cancelled", "trialing", etc.
  stripePlanName       String?   // Human-readable plan name from Stripe (e.g., "Growth 4")                                                                  
  stripeCurrentPeriodEnd DateTime? // When the current billing period ends                                                                                   
                                                                                                                                                             
  Run npx prisma migrate dev to create the migration.                                                                                                        
                                                                                                                                                             
  Task 4: Create Stripe Webhook Endpoint                                                                                                                     
                                                                                                                                                           
  Create /api/webhooks/stripe/route.ts — a POST endpoint that:                                                                                               
  
  1. Reads the raw request body                                                                                                                              
  2. Verifies the Stripe signature using STRIPE_WEBHOOK_SECRET                                                                                             
  3. Handles these event types:                                                                                                                              
  
  customer.subscription.created and customer.subscription.updated:                                                                                           
  - Extract customer (Stripe customer ID), status, items.data[0].price.product (product name), and current_period_end                                      
  - Look up the User by stripeCustomerId                                                                                                                     
  - If no user found, look up the Stripe customer by ID to get their email, then match by email on the User model                                          
  - If a user is found: update stripeCustomerId (if not already set), stripeSubscriptionId, subscriptionStatus, stripePlanName (fetch the product name from  
  Stripe), stripeCurrentPeriodEnd                                                                                                                            
  - If status is "active", also update serviceTier to match the Stripe product (map product names to the ServiceTier enum: Production variants →             
  editing_2/editing_4, Growth variants → mastery_2/mastery_4, etc.). If the product doesn't map cleanly, leave serviceTier unchanged.                        
                                                                                                                                                             
  customer.subscription.deleted:                                                                                                                           
  - Find User by stripeCustomerId or by customer email                                                                                                       
  - Set subscriptionStatus to "cancelled", clear stripeSubscriptionId                                                                                      
  - Optionally downgrade serviceTier to "foundations"                                                                                                        
                                                                                                                                                             
  4. Return 200 for all handled events, 200 for unhandled events (don't error on events we don't care about)                                                 
                                                                                                                                                             
  Important: This route must NOT use the Next.js body parser — Stripe signature verification requires the raw body. Use the standard pattern for Next.js App 
  Router Stripe webhooks.                                                                                                                                    
                                                                                                                                                             
  Task 5: Create Stripe Utility                                                                                                                            

  Create src/lib/stripe.ts:                                                                                                                                  
  
  import Stripe from 'stripe';                                                                                                                               
                                                                                                                                                           
  export const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, {
    apiVersion: '2024-12-18.acacia', // use latest stable
  });                                                                                                                                                        
  
  Task 6: Backfill Script                                                                                                                                    
                                                                                                                                                           
  Create scripts/stripe-backfill.ts that:                                                                                                                    
  
  1. Fetches all users from the database                                                                                                                     
  2. For each user with an email, searches Stripe customers by email (stripe.customers.list({ email }))                                                    
  3. If a matching Stripe customer is found with an active subscription:                                                                                     
    - Updates the user's stripeCustomerId, stripeSubscriptionId, subscriptionStatus, stripePlanName, stripeCurrentPeriodEnd
  4. Logs a summary: how many users matched, how many had active subscriptions, how many couldn't be matched                                                 
                                                                                                                                                             
  This is a one-time script to populate existing members. Run with: npx tsx scripts/stripe-backfill.ts                                                       
                                                                                                                                                             
  Important Notes                                                                                                                                            
                                                                                                                                                           
  - The webhook endpoint must be publicly accessible for Stripe to reach it. In Replit, the dev URL works for testing. For production, the webhook URL will  
  need to be registered in the Stripe dashboard.                                                                                                           
  - Use Canadian spelling in any user-facing text.                                                                                                           
  - Don't break any existing functionality — this is purely additive.
