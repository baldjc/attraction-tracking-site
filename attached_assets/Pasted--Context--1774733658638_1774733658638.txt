  Task 4: Manual Stripe Linking Modal                                                                                                                        
                                                                                                                                                             
  Create a modal that appears when an admin clicks "Link Stripe Customer" on a member without a stripeCustomerId. The modal should:                          
                                                                                                                                                           
  1. Show a search input where the admin can type a name or email                                                                                            
  2. Search Stripe customers via a new API route: GET /api/admin/members/stripe-search?q=searchterm                                                        
  3. Display matching Stripe customers with: name, email, and active subscription (if any)                                                                   
  4. Admin clicks "Link" on the correct customer → calls PUT /api/admin/members/[id]/stripe-link with the Stripe customer ID                               
  5. The API route updates the user's stripeCustomerId and fetches + saves their current subscription data                                                   
  6. Modal closes, member data refreshes                                                                                                                     
                                                                                                                                                             
  Task 5: API Routes                                                                                                                                         
                                                                                                                                                             
  Create the following new API routes:                                                                                                                       
                                                                                                                                                             
  GET /api/admin/members/stripe-search                                                                                                                     
  - Query param: q (search string)                                                                                                                         
  - Uses stripe.customers.search({ query: name~"${q}" OR email~"${q}" }) to search Stripe                                                                    
  - Returns array of { id, name, email, subscription: { planName, status } | null }      
  - Admin-only auth check                                                                                                                                    
                                                                                                                                                           
  PUT /api/admin/members/[id]/stripe-link                                                                                                                    
  - Body: { stripeCustomerId: string }                                                                                                                       
  - Fetches the Stripe customer's active subscriptions                                                                                                       
  - Updates the User record with stripeCustomerId, stripeSubscriptionId, subscriptionStatus, stripePlanName, stripeCurrentPeriodEnd                          
  - Admin-only auth check                                                                                                                                  
                                                                                                                                                             
  PUT /api/admin/members/[id]/stripe-unlink                                                                                                                  
  - Clears all Stripe fields on the User record (stripeCustomerId, stripeSubscriptionId, subscriptionStatus, stripePlanName, stripeCurrentPeriodEnd all set  
  to null)                                                                                                                                                   
  - Admin-only auth check

  Also add query param support for filtering by subscription status: ?subscriptionStatus=active etc.
