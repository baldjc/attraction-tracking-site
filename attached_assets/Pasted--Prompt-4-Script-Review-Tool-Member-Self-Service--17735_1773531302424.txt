  Prompt 4 — Script Review Tool (Member Self-Service)                                                                                    
                                                                                                                                         
  Build a Script Review tool that lets members paste a script or transcript and get instant AI feedback before they publish.             
                                                            
  Add "Script Review" to the member sidebar navigation (with a pencil/document icon). Place it between "My Scores" and "My Links".

  SCRIPT REVIEW PAGE (/member/script-review):

  Layout:

  1. **Header** — "Script Review" with subtext: "Paste your script or transcript below to get scored against the 16 Attraction principles
   before you publish."

  2. **Input Form:**
     - Video Title (required) — text input with placeholder: "What's your working title?"
     - Script/Transcript (required) — large text area, minimum 10 rows, with placeholder: "Paste your full script or transcript here..."
     - "Review My Script" button (brand blue #3dc3ff)

  3. **While processing** — Show a progress indicator: "Analysing your script..." with a spinner. This will take 15-30 seconds.

  4. **Results Display** (appears below the form after processing):

     a. **Script Score callout** — colour-coded:
        "Script Attraction Score: X.X / 10"
        Below it, if the member has a baseline audit, show: "Your baseline: X.X — this script scores {higher/lower} by X.X"

     b. **4 Dimension Scores** — horizontal cards:
        🎯 Channel Strategy: X.X
        🎬 Content Impact: X.X
        🤝 Viewer Connection: X.X
        📈 Lead Generation: X.X
        Colour-coded: blue for 7+, amber for 5-6.9, crimson for below 5

     c. **16-Principle Scorecard** — Same grouped table as audit reports:
        Principle | This Script | Baseline | Δ
        Colour-coded scores and deltas
        If no baseline exists, hide the Baseline and Δ columns

     d. **Three Ideas for Improvement** — Same format as single video audit:
        Each idea has:
        - Principle name and score
        - What's happening in the script (with a quote from their text)
        - What to change (with a concrete rewrite example using THEIR content)
        - Foundations lesson reference

     e. **What's Working** — Green callout with 3-4 strengths found in the script

     f. **Quick Win** — Yellow callout: one specific thing to change before recording

  5. **"Review Again" button** — Clears the results so they can edit their script and resubmit. Keep the script text in the textarea so
  they can modify it without retyping.

  6. **"Save Review" button (optional)** — Saves the review to the database for their reference. NOT counted as an official audit — saved
   separately.

  DATABASE — Add a new table:

  script_reviews:
  - id (UUID)
  - member_id (FK to users)
  - video_title (string)
  - script_text (text)
  - scores (JSON — same 16-principle format as audits)
  - overall_score (decimal)
  - report_content (JSON — dimension scores, strengths, improvements, quick win)
  - created_at

  SCRIPT REVIEW HISTORY:
  Below the input form, show a list of past saved reviews:
  - Title | Score | Date | "View" button
  - Clicking View shows the full results again
  - Keep the last 10 reviews per member

  API ENDPOINT — POST /api/script-review:
  - Accepts: { videoTitle, scriptText }
  - Gets the logged-in member's ID from session
  - Sends the script to Claude (same model: claude-sonnet-4-20250514) with a modified scoring prompt

  The scoring prompt for script reviews is similar to the single video audit prompt but adapted:
  - It's scoring a SCRIPT, not a published video — so principles like Consistency, Show Don't Tell, and view count references don't apply
   the same way
  - Score Show Don't Tell based on visual cues written into the script (e.g., "[show chart]", "as you'll see on screen", B-roll notes)
  - Score Consistency as N/A or based on the member's publishing history from their baseline
  - All other principles score normally from the text content
  - The response format should match the single video audit format

  IMPORTANT: Script reviews are NOT audits. They:
  - Do NOT appear in the member's audit history
  - Do NOT affect their official score or score trend
  - Do NOT show up on the admin audits page
  - Are private to the member (admin cannot see them unless we add that later)
  - Are a self-service coaching tool
