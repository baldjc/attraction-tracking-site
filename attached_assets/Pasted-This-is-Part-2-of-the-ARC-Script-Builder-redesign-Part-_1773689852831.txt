=== CHANGE 6: REWRITE ARC SCRIPT BUILDER API ===

Completely rewrite src/app/api/ai-tools/arc-script-builder/route.ts to support two modes:

MODE 1: step === "summarize"
- Accepts: { step: "summarize", researchText: string, title: string, talkingPoints?: string }
- Sends all research text to Claude with a prompt to summarize into a structured brief (~1,500-2,000 tokens)
- The summary prompt should extract: key facts/stats, main arguments, client pain points, story angles, credibility data, notable quotes, what the avatar hears from other sources
- Logs usage via logUsage() from ai-tool-cost.ts
- Checks cost cap first — returns 429 with { error: "monthly_cap_reached", resetsAt } if exceeded
- Returns: { summary: string, usage: { inputTokens, outputTokens } }

MODE 2: step === "chat"
- Accepts: { step: "chat", messages: Array<{ role, content, researchSummary? }> }
- The first message in the array may have a researchSummary field — extract it and inject into the system prompt as {{RESEARCH_SUMMARY}}
- Strip researchSummary from messages before sending to Claude (it goes in the system prompt, not as a message)
- Uses STREAMING: client.messages.stream() instead of client.messages.create()
- Returns a Server-Sent Events stream (text/event-stream):
  - data: {"type":"text","text":"..."} for each text chunk
  - data: {"type":"done","sectionData":{...},"costCapWarning":null} when complete
- After stream completes, log usage and check cost cap percentage
- Checks cost cap before processing — returns 429 if exceeded

SYSTEM PROMPT for the chat mode (this is the default, admin can edit via AppSetting "prompt_arc_script_builder"):

The AI walks through 7 ARC sections in order, one at a time:

1. RESEARCH SUMMARY — Present structured brief. Check for gaps:
   - At least one stat/data point
   - At least one client story or personal experience
   - Awareness of what avatar hears from other sources
   - Enough talking points for planned insights
   Flag missing items, suggest what to research. Also check for even-numbered listicles and suggest odd numbers (3, 5, 7, 9).

2. OPENING (~20-25 sec) — Generate 2-3 opening options from the 4 ARC patterns (Contradiction, Confirmation, Empathy, Stakes). Write word-for-word scripts.

3. CREDIBILITY — Draft using credentials from avatar profile. Note what was pulled in.

4. INSIGHTS (VALUE LOOPS) — Generate What → Why → When → Story → Connection loops from research. Add curiosity bridges (And → But → Therefore).

5. CLOSING — Draft closing with lead magnet mentions (3 total across script). Note connection phrases and values placement.

6. LEAD MAGNET BRAINSTORM — Generate 2-3 ideas using this principle: "Marketing is the continuation of the thought the client is having, and the lead magnet shows up as that continuation of thought." Must be specific to this video, not generic.

7. FINAL SCRIPT — Assemble everything, run 11-item checklist.

The AI pulls avatar credentials, values, and interests SILENTLY and notes what it used. If no avatar saved, notes "build your avatar first."

Each AI message must end with:
<SECTION_DATA>
{"currentSection": "research_summary|opening|credibility|insights|closing|lead_magnets|final_script", "sectionApproved": true|false}
</SECTION_DATA>

When a section is approved, set currentSection to the NEXT section being moved to. sectionApproved true means the previous section was just approved.

The system prompt uses these placeholders (replaced at runtime):
- {{MEMBER_AVATAR}} — avatar profile JSON + name
- {{CONTENT_THEMES}} — content themes JSON
- {{BASELINE_SCORES}} — latest baseline audit scores JSON
- {{RESEARCH_SUMMARY}} — the structured research brief

Check AppSetting "prompt_arc_script_builder" first; fall back to hardcoded default.

Use claude-sonnet-4-20250514 for both modes. max_tokens: 2048 for summarize, 4096 for chat.

=== CHANGE 9: CHAT PHASE COMPONENT ===

Create src/components/ai-tools/ArcScriptChatPhase.tsx

This is the chat interface shown after the upload phase completes. It follows the EXACT same chat pattern as the Avatar Architect (src/app/member/ai-tools/avatar-architect/page.tsx) — same message bubbles, same input area, same scroll behaviour.

Key differences from Avatar Architect:
1. Progress bar at top (ArcProgressBar component)
2. Inline approval card when clicking a completed section (expandable/collapsible card showing approved text)
3. STREAMING responses — uses EventSource/ReadableStream reader instead of awaiting JSON:

   - Update the last assistant message incrementally as text arrives
   - After "done" event, process sectionData and costCapWarning
4. Cost cap warning banners (amber at 75%, red at 90%, locked at 100%)
5. Turn limit: 20 turns (user + assistant combined). After 20, show limit message.
6. Final script actions: Copy, Save, Build Another Script buttons (shown when currentSection === "final_script")

Props: { initialData: { title, talkingPoints, researchSummary }, onReset: () => void }

Section tracking:
- Parse <SECTION_DATA> from AI responses (strip from display text)
- When sectionApproved is true, mark the previous section as complete
- Store a snippet of the approved content for each section (for the inline approval card)

Save script: POST to /api/ai-tools/save-script with the same scriptOutline JSON shape as current (opening, credibility, lead_magnet_1, insights array, etc). Assemble from approved sections.
