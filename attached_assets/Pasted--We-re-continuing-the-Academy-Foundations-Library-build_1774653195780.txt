We're continuing the Academy / Foundations Library build on the "feature/academy-foundations-library" branch. Make sure you're on that branch. This is Part 3 of 4 — admin course manager.

=== ADMIN API ROUTES (13 routes) ===

All routes require admin role. Use resolveUserFromSession() and check role.

1. GET /api/admin/academy/sections — list all sections (published and unpublished) with lesson counts
2. POST /api/admin/academy/sections — create a new section (title, description, sortOrder, published)
3. PUT /api/admin/academy/sections/[id] — update section
4. DELETE /api/admin/academy/sections/[id] — delete section. If section has lessons, return error with message "This section has X lessons. Delete lessons first."

5. POST /api/admin/academy/lessons — create a new lesson (all lesson fields + sectionId)
6. PUT /api/admin/academy/lessons/[id] — update lesson details
7. DELETE /api/admin/academy/lessons/[id] — delete lesson. If member progress or responses exist, return warning but allow with ?force=true query param.

8. PUT /api/admin/academy/reorder — body: { sections: [{ id, sortOrder }], lessons: [{ id, sortOrder }] } — batch update sort orders

9. GET /api/admin/academy/lessons/[id]/workbook-fields — list all workbook fields for a lesson
10. POST /api/admin/academy/lessons/[id]/workbook-fields — add a workbook field
11. PUT /api/admin/academy/workbook-fields/[id] — update a workbook field
12. DELETE /api/admin/academy/workbook-fields/[id] — delete a workbook field (warn if responses exist, allow with ?force=true)

13. GET /api/admin/academy/member-progress/[userId] — get a member's full course progress:
    - Overall: % complete, lessons finished, total lessons
    - Per section: completed/total
    - Workbook: fields filled / total fields
    - Homework: items completed / total items
    - Last lesson viewed + date
