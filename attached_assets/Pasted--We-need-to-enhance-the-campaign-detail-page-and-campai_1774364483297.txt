  We need to enhance the campaign     
  detail page and campaigns list page
  with source-type-aware metrics, a   
  geographic click heat map, and a   
  location breakdown table.           
                                      
  === STEP 1: SOURCE-TYPE-AWARE STATS 
  BAR ===                           
                                      
  The stats shown on both the campaign
   detail page AND the campaign cards
  on the main campaigns list page
  should change based on the
  campaign's sourceType.   

  Email Newsletter campaigns:         
  - Show: Clicks (total), Unique
  Clicks (count of distinct IP        
  addresses)                        
  - Hide: Leads, Conversion Rate,   
  Views                    
  - The existing analytics line chart
  should also hide the Leads line for
  email campaigns          

  YouTube campaigns:                  
  - Show: Views (sum of YouTube video
  views across all links), Clicks,    
  Leads, Conversion Rate (leads ÷   
  clicks)                           
  - Views should also appear on the
  campaign card on the main campaigns
  list page                
                           
  Google Ads / Other campaigns:
  - Show: Clicks, Leads, Conversion   
  Rate                             
                                      
  Keep the same card/stat styling   
  that's already in use. Just       
  conditionally render the right
  metrics based on sourceType.
                           
  === STEP 2: UNIQUE CLICKS
  CALCULATION ===          
                           
  Add a unique clicks count for Email
  Newsletter campaigns:               
  - Unique clicks = COUNT(DISTINCT
  ipAddress) from the Click table for 
  that campaign (or filtered by      
  specific tracking link)           
  - This is a query-time aggregation,
  no new database columns needed
  - Used in the stats bar and location
   table for email campaigns only
                           
  === STEP 3: CLICK HEAT MAP WITH
  LEAFLET ===              
                           
  Add a geographic click map to EVERY
  campaign detail page (all source
  types, not just email).             
                           
  Install these packages:             
  - leaflet                         
  - react-leaflet                   
  - leaflet.markercluster (for
  clustering dense click areas)       
  - @types/leaflet (dev dependency)
                                      
  Create a reusable ClickMap        
  component:                        

  1. Use OpenStreetMap tiles (free, no
   API key): https://{s}.tile.openstre
  etmap.org/{z}/{x}/{y}.png
  2. Use dynamic import with
  next/dynamic and ssr: false (Leaflet
   needs window/document)  
  3. Plot click locations as circle
  markers on the map
  4. Use marker clustering            
  (leaflet.markercluster) so dense
  areas show a count bubble instead of
   overlapping dots                 
  5. Auto-fit the map bounds to the 
  actual click data points. If all
  clicks are in Calgary, zoom to
  Calgary. If they span the country,
  zoom out to fit all points.
  6. Empty state: if no clicks have   
  geolocation data, show a 
  muted/greyed map with "No location  
  data yet" text centred            
                                      
  Important data source difference:
  - For Email Newsletter campaigns:   
  plot UNIQUE clicks only (one marker
  per unique IP address, using the  
  coordinates from the most recent
  click for that IP)       
  - For all other campaign types: plot
   ALL clicks (every click record gets
   a marker)               

  Add a link filter dropdown above the
   map (right-aligned):
  - Default: "All Links"              
  - Options: "All Links" + each     
  tracking link name in the campaign
  - Changing the dropdown re-filters
  both the map markers AND the
  location table below

  total clicks
