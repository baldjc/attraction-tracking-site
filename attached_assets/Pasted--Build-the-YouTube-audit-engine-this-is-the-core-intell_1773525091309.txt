  ## STEP 1: YouTube Data Fetching (src/lib/youtube.ts)

  Create a YouTube data service:

  1. **Channel info:**
     - GET
  https://www.googleapis.com/youtube/v3/channels?part=snippet,brandingSettings,contentDetails&forHandle={handle}&key={YOUTUBE_API_KEY}
     - Extract: channel banner URL from brandingSettings.image.bannerExternalUrl
     - Extract: uploads playlist ID from contentDetails.relatedPlaylists.uploads

  2. **Video list (get latest 5 long-form):**
     - GET
  https://www.googleapis.com/youtube/v3/playlistItems?part=snippet&playlistId={uploadsPlaylistId}&maxResults=15&key={YOUTUBE_API_KEY}
     - Then for each video: GET
  https://www.googleapis.com/youtube/v3/videos?part=snippet,contentDetails,statistics&id={videoId}&key={YOUTUBE_API_KEY}
     - Filter OUT Shorts (duration < 60 seconds — parse ISO 8601 duration from contentDetails.duration)
     - Take the 5 most recent long-form videos
     - For each video, collect: title, duration, upload date, view count, video ID

  3. **Transcripts:**
     - npm install youtube-transcript
     - import { YoutubeTranscript } from 'youtube-transcript'
     - const transcript = await YoutubeTranscript.fetchTranscript(videoId)
     - Join segments into plain text with approximate timestamps
     - If transcript fails for a video, note it but continue — score based on available data

  ## STEP 2: AI Scoring Engine (src/lib/audit-engine.ts)

  Create the scoring engine that sends transcripts to Claude for analysis.

  Use the Anthropic SDK: npm install @anthropic-ai/sdk

  The scoring prompt scores against 16 principles (0-10 each):

  1. Avatar Clarity — Is there ONE clear audience?
  2. Themes Over Topics — Are there 2-4 repeatable content themes?
  3. ARC Attention — How strong are the opening hooks?
  4. ARC Revelation — Are there genuine, unique insights?
  5. ARC Connection — Is there emotional resonance and trust?
  6. Title Frameworks — Do titles use proven click patterns?
  7. Approve the Click — Do first 30 seconds match the title promise?
  8. Lead Magnet System — Is there a free resource mentioned 3x per video?
  9. Curiosity Bridges — Do transitions pull viewers forward?
  10. Show Don't Tell — Are there visual proof elements?
  11. Values Peppering — Does the viewer learn about the person beyond real estate?
  12. Connection Language — Are there phrases that make the avatar feel spoken to?
  13. Story Proof — Are there client stories with names, situations, stakes, outcomes?
  14. Grade 5 Language — Could a 10-year-old follow along?
  15. Binge Architecture — Are there cross-references to other videos?
  16. Consistency — How regular is the publishing schedule?

  Store the AI scoring prompt as a configurable text field in a settings table in the database (seed it with the default prompt). Load it
   from the database when running audits so it can be refined from the Settings page without code changes.

  Send to Claude (model: claude-sonnet-4-20250514) with the scoring prompt as system message, and the transcripts + video metadata as
  user message.

  Claude must return a JSON response with this structure:
  {
    "scores": {
      "avatar_clarity": { "score": 5.5, "evidence": "..." },
      "themes_over_topics": { "score": 4.0, "evidence": "..." },
      "arc_attention": { "score": 0.0, "evidence": "..." },
      "arc_revelation": { "score": 0.0, "evidence": "..." },
      "arc_connection": { "score": 0.0, "evidence": "..." },
      "title_frameworks": { "score": 0.0, "evidence": "..." },
      "approve_the_click": { "score": 0.0, "evidence": "..." },

      "curiosity_bridges": { "score": 0.0, "evidence": "..." },
      "show_dont_tell": { "score": 0.0, "evidence": "..." },
      "values_peppering": { "score": 0.0, "evidence": "..." },
      "connection_language": { "score": 0.0, "evidence": "..." },
      "story_proof": { "score": 0.0, "evidence": "..." },
      "grade_5_language": { "score": 0.0, "evidence": "..." },
      "binge_architecture": { "score": 0.0, "evidence": "..." },

  - Below 5.0 → red

  ## STEP 3: Audit Job Processing (src/app/api/audits/run/route.ts)

  Create POST /api/audits/run that:
  1. Accepts: { memberId, auditType: "baseline" | "monthly" | "single_video", videoId?: string }
  2. Creates an audit_job record with status "queued"
  3. Returns the job ID immediately
  4. Processes asynchronously:

  Job steps:
  a. Status → "downloading" — fetch channel data (banner, video list)
  b. For baseline: fetch transcripts for 5 most recent long-form videos
  c. For monthly: fetch transcripts for videos published SINCE the last audit date. If fewer than 2 new videos, return error: "Not enough
   new content for a meaningful monthly audit"
  d. For single_video: fetch transcript for the specified video only
  e. Status → "analysing" — send to Claude scoring engine
  f. Status → "generating" — build report content
  g. Save audit record to database with all 16 scores, overall score, report content JSON, videos analysed
  h. For monthly: also store baseline scores and previous month scores in the report_content JSON for comparison
  i. Status → "complete"
  j. If any step fails: status → "failed" with error message

  Job status endpoint: GET /api/audits/jobs/[jobId]
  - Returns current status, progress message, and audit_id when complete
  - Frontend polls every 5 seconds

  ## STEP 5: Report Display Pages

  Build 3 report page types. These should be polished, branded, and match the app's colour palette.

  ### BASELINE REPORT (/admin/audits/[auditId] with type=baseline)
  Login required (admin or the member themselves).

  Layout:
  1. YouTube channel banner (full width)
  2. Blue callout: "Attraction by Video — Baseline Audit" with @handle
  3. Overall score callout (colour-coded): "Your Attraction Score: X.X / 10"
  4. Full 16-principle scorecard table: Principle | Score (colour-coded)
  5. Videos analysed table: Title | Duration | Date | Views
  6. Video-by-video breakdown — for each video: Opening analysis, Insights analysis, Connection analysis (from Claude's video_breakdowns)
  7. Learning Path — principles below 7 mapped to lessons:
     Avatar Clarity → 1.1 + 1.2, Themes → 1.3, Lead Magnet → 1.4, Values Peppering → 2.1, Connection Language → 2.2, ARC Attention → 2.5
  + 2.5a + 3.2, ARC Revelation → 2.5, ARC Connection → 2.2 + 2.5, Curiosity Bridges → 2.5, Story Proof → 2.5, Show Don't Tell → 3.3,
  Approve the Click → 4.1 + 2.5, Title Frameworks → 4.2, Binge Architecture → 1.3, Grade 5 Language → N/A, Consistency → 1.3 + 2.4
  8. Q&A Topics (flagged items for coaching calls — always flag: Lead Magnet, Avatar, Connection Language, Approve the Click, Curiosity
  Bridges)
  9. What's Working — 3-4 strengths with evidence
  10. One-Sentence Diagnosis: "{Name} has {strength} — but {core gap}."
  11. Three Biggest Gaps with Current vs Improved examples
  12. Footer: "Prepared for {Name} by Jared Chamberlain ~ Founder of Attraction by Video"

  ### MONTHLY REPORT (/admin/audits/[auditId] with type=monthly)
  Login required.

  Layout:
  1. Blue callout with month/year
  2. Channel banner
  3. Progress summary callout: This month score, Δ from baseline, Δ from last month (with arrows ↑↓→)
  4. Score comparison table: 16 principles × Baseline | Last Month | This Month | Δ Baseline columns
     - Green cell: improved 1+ from baseline
     - Yellow cell: improved 0.5
     - Red cell: declined from baseline
  5. What Improved This Month — specific principles with video evidence
  6. Videos analysed this month (table)
  7. Video-by-video breakdown
  8. Remaining Gaps — principles still below 7, ordered by priority
  9. Updated Learning Path (removes lessons for principles at 7+)
  10. What's Working (maintained + new strengths)
  11. Purple callout: one-sentence coaching summary
  12. Footer

  ### SINGLE VIDEO REPORT (/admin/audits/[auditId] with type=single_video)
  Login required.

  Layout:
  1. Channel banner
  2. Blue callout: "Attraction by Video — Single Video Audit"
  3. Score callout: "Video Attraction Score: X.X / 10" with video title, duration, date
  4. Scorecard table: Principle | Baseline | This Video | Δ (colour-coded deltas)
  5. Video Deep Dive organised by PHASE (not principle-by-principle):
     - **Opening (0:00–~1:00):** ARC Attention + Approve the Click + Lead Magnet first mention
     - **Body:** ARC Revelation + Show Don't Tell + Curiosity Bridges
     - **Connection & Voice:** ARC Connection + Connection Language + Values Peppering + Story Proof + Grade 5 Language
     - **Channel Strategy:** Avatar Clarity + Themes + Binge Architecture + Consistency + Lead Magnet full assessment
  6. What's Working — green callout with timestamps
  7. Three Ideas for Improvement — each with: principle, score, what happened (with timestamp), what to do instead (with rewrite
  example), Foundations lesson reference
  8. Quick Wins callout (yellow): one immediately actionable thing
  9. Q&A Prep callout (blue): 2-3 topics with what to bring to the call
  10. Purple callout: one-sentence coaching summary
  11. Footer
