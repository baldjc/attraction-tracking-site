## WHERE THE CHANGES NEED TO HAPPEN

There are **3 separate places** that need changes. If you only change the UI, the scores won't change. All 3 must be updated:

1. **The scoring prompt string** — the actual text sent to the Claude API when running an audit. This is where the AI reads the rubric. If the rubric text doesn't change here, the AI will keep scoring wrong.

2. **The score calculation code** — the server-side code that takes the 16 raw scores from the AI response and calculates the weighted overall score. This is where Show Don't Tell needs to be excluded.

3. **The UI/report display** — labels, formatting, N/A states. (Some of this was already done.)

## Fix 7: Monthly Audits — Always Pull Last 5 Videos, Not Just New Ones

### The Problem
Monthly audits currently only pull videos published since the last audit. If a member published 2 new videos in a month, the AI only sees 2 videos — not enough to judge Consistency, Themes, Binge Architecture, or any pattern-based principle. This is why Consistency scores are wrong (1–2 videos gives almost no upload date data).

### How Monthly Audits Should Work

**Step 1: Check eligibility**
- Look up the date of the member's most recent audit (baseline or monthly)
- Check if the member has published **at least 1 new video** since that date
- If NO new videos → **skip this member** (don't run the audit). Show a message like "No new content since last audit" in the audit queue
- If YES → proceed to Step 2

**Step 2: Pull the last 5 videos**
- Pull the member's **5 most recent long-form videos** (same as a baseline audit)
- NOT just the videos since the last audit — always the last 5
- This gives the AI enough data to assess patterns (Consistency, Themes, Binge Architecture, etc.)

**Step 3: Score normally**
- Send all 5 videos to the Claude API with the monthly audit prompt
- The monthly prompt already includes baseline scores for comparison — that's fine
- The AI scores the current state of the channel based on the most recent 5 videos

### Why This Is Better
- Consistency: 5 videos with upload dates = accurate cadence calculation
- Themes: 5 videos shows whether content pillars are being maintained
- Binge Architecture: 5 videos shows cross-referencing patterns
- The "at least 1 new video" gate prevents re-auditing when nothing has changed
- This matches how a human coach would assess — look at the latest work, not just the delta

### What to Change in Code

**Find the monthly audit video fetching logic** (likely in the audit engine or a YouTube data service). It currently does something like:

```javascript
// OLD: Only fetch videos after last audit date
const videos = await getVideosAfterDate(channelId, lastAuditDate);
```

**Change to:**

```javascript
// NEW: Check for new content, then always fetch last 5
const lastAuditDate = await getLastAuditDate(memberId);
const latestVideo = await getLatestVideo(channelId);

// Gate: at least 1 new video since last audit
if (latestVideo.publishDate <= lastAuditDate) {
  // Skip — no new content
  return { status: 'skipped', reason: 'No new videos since last audit' };
}

// Always pull last 5 videos regardless of audit date
const videos = await getLatestVideos(channelId, 5);
```

**For the "Run All Monthly Audits" batch button:**
- Skip members with no new videos (don't create a failed/empty audit)
- Show which members were skipped and why in the batch progress UI
