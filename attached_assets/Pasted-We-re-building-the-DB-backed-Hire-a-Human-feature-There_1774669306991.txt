=== STEP 1: PRISMA SCHEMA ===

Add these two models to prisma/schema.prisma:

model ServiceCategory {
  id            String   @id @default(uuid())
  name          String
  slug          String   @unique
  description   String?
  icon          String   @default("PuzzlePieceIcon")
  accentColour  String   @default("blue")
  sortOrder     Int      @default(0)
  published     Boolean  @default(true)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  packages      ServicePackage[]

  @@map("service_categories")
}

model ServicePackage {
  id                String   @id @default(uuid())
  categoryId        String
  name              String
  price             String
  priceNote         String?
  badge             String?
  subtitle          String?
  features          Json     @default("[]")
  highlightFeatures Json?
  stripeUrl         String?
  sortOrder         Int      @default(0)
  published         Boolean  @default(true)
  createdAt         DateTime @default(now())
  updatedAt         DateTime @updatedAt
  category          ServiceCategory @relation(fields: [categoryId], references: [id])

  @@map("service_packages")
}

Run: npx prisma migrate dev --name add-service-categories-packages

=== STEP 3: API ROUTES ===

Create these API routes:

--- Member API ---

GET /api/member/hire/categories
- Auth required (session check)
- Returns all published categories with their published packages
- Ordered by category.sortOrder, then package.sortOrder
- Response shape:
  {
    categories: [
      {
        id, name, slug, description, icon, accentColour, sortOrder,
        packages: [
          { id, name, price, priceNote, badge, subtitle, features, highlightFeatures, stripeUrl, sortOrder }
        ]
      }
    ]
  }


--- Admin API ---

GET /api/admin/hire/categories
- Admin auth required
- Returns ALL categories (including unpublished) with ALL packages (including unpublished)
- Same shape as member API but with published field included

POST /api/admin/hire/categories
- Create a new category
- Body: { name, slug, description?, icon?, accentColour? }
- Auto-set sortOrder to max + 1
- Returns the created category

PUT /api/admin/hire/categories/[id]
- Update category fields
- Body: any subset of { name, slug, description, icon, accentColour, sortOrder, published }
- Returns updated category

DELETE /api/admin/hire/categories/[id]
- Delete a category
- FAIL with 400 if it has any packages (force admin to delete packages first)
- Returns { success: true }

POST /api/admin/hire/packages
- Create a new package
- Body: { categoryId, name, price, priceNote?, badge?, subtitle?, features, highlightFeatures?, stripeUrl? }
- Auto-set sortOrder to max within category + 1
- Returns created package

PUT /api/admin/hire/packages/[id]
- Update package fields
- Body: any subset of { categoryId, name, price, priceNote, badge, subtitle, features, highlightFeatures, stripeUrl, sortOrder, published }
- Returns updated package

DELETE /api/admin/hire/packages/[id]
- Delete a package
- Returns { success: true }

PUT /api/admin/hire/reorder
- Reorder categories OR packages
- Body: { type: "category" | "package", items: [{ id, sortOrder }] }
- Updates sortOrder for each item
- Returns { success: true }
