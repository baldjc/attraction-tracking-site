  === CHANGE 2: NEW API ROUTES FOR LIVE CALLS ===     
                                                      
  Create two new API routes:                          
                                                 
  1. GET /api/member/academy/live-calls               
   
  Auth: any logged-in member (use the same auth       
  pattern as existing member routes).
                                                      
  Returns all QACalls with status "processed",        
  enriched with aggregated data from their       
  KnowledgeBaseEntries.                               
                  
  Logic:                                         
  - Fetch all QACalls where status = "processed",
  ordered by callDate desc                            
  - For each call, fetch its KnowledgeBaseEntries
  where sourceType = "qa_call" AND sourceId = call.id 
  AND status = "approved"                             
  - For privacy: only include entries where      
  isGeneralTeaching = true OR memberId = current      
  user's ID                                           
  - Aggregate: collect all unique principles across
  the call's entries, take the first 1-2 general      
  teaching entries' summaries as the call summary,    
  count total moments                             
  - Group results by month (use callDate)             
                  
  Response shape:                                     
  {
    months: [                                         
      {           
        label: "March 2026",                          
        calls: [  
          {                                      
            id: string,
            title: string,
            callDate: string (ISO),                   
            duration: number | null,
            fathomShareUrl: string,                   
            summary: string,                          
            principles: string[],                
            momentCount: number                       
          }       
        ]                                        
      }
    ]
  }

  2. GET /api/member/academy/live-calls/[callId]      
   
  Auth: any logged-in member.                         
                  
  Returns full detail for a single Q&A call.          
   
  Logic:                                              
  - Fetch the QACall by id (must have status =
  "processed")                                   
  - Fetch all KnowledgeBaseEntries for this call
  (sourceType = "qa_call", sourceId = callId, status =
   "approved")                                        
  - Privacy filter: only include entries where
  isGeneralTeaching = true OR memberId = current user 
  ID. Admin users see everything.                     
  - Redact other member names in searchableText for
  non-admin users (use the same redaction pattern from
   the existing /api/member/resources route — find the
   redactForMember function and reuse it)             
  - Collect all unique principles from the entries    
  - Find related Foundations Library lessons: query
  CourseLesson records (from the academy_lessons 
  table) whose principleTags JSON array contains ANY  
  of the call's principles. Include section title via
  the CourseSection relation.                         
                  
  Response shape:                                
  {
    call: {
      id: string,
      title: string,
      callDate: string (ISO),
      duration: number | null,
      fathomShareUrl: string,                         
      principles: string[]
    },                                                
    moments: [    
      {                                          
        id: string,
        subTopic: string,
        summary: string,
        principles: string[],
        timestampStart: number | null,                
        timestampEnd: number | null,
        isGeneralTeaching: boolean,                   
        isMine: boolean
      }                                               
    ],
    relatedLessons: [                                 
      {           
        id: string,                              
        title: string,
        slug: string,
        sectionTitle: string,
        sectionSlug: string,                          
        principleTags: string[]
      }                                               
    ]             
  }
